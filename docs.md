Comentix - требования к сервису игры Last Comment Wins
1. Общая концепция проекта
1. Необходимо разработать веб‑сервис игры Last Comment Wins под брендом Comentix для обычных опубликованных видео YouTube (не для live‑стримов).
2. Игра проводится через отдельную веб‑страницу сервиса Comentix, ссылка на которую размещается в описании YouTube‑видео.
3. На странице игры пользователи должны видеть текущий статус игры, таймер, участников и призовые категории.
4. Сервис должен автоматически отслеживать комментарии под выбранным YouTube‑видео и использовать их как действия участников игры.


2. Старт игры
5. Игра не должна запускаться по вручную заданной дате и времени.
6. После настройки в админ‑панели игра переводится в режим ожидания первого комментария.
7. Игра должна автоматически запускаться после первого валидного комментария.
8. Первый валидный комментарий запускает основной таймер игры, фиксирует фактическое время начала игры и считается первым участником.
9. До первого валидного комментария на странице игры должен отображаться статус ожидания.
3. Таймер игры
10. Базовая длительность таймера должна настраиваться в админ‑панели.
11. После первого валидного комментария запускается таймер на заданное время.
12. Каждый следующий валидный комментарий должен увеличивать таймер игры.
13. Величина увеличения таймера должна настраиваться.
14. Примеры увеличения: +30 секунд, +1 минута, +5 минут.
15. Таймер должен отображаться на странице игры и обновляться автоматически.
16. Игра должна завершаться, если после последнего валидного комментария прошло заданное время периода тишины.
17. Период тишины должен настраиваться (например 1, 5 или 10 минут).


4. Правила валидного комментария
18. В игре должны учитываться только валидные комментарии.
19. Требования к валидности комментария должны настраиваться.
20. Комментарий должен соответствовать минимальным требованиям содержательности.
21. Возможные параметры проверки: минимальное количество слов, минимальное количество символов, минимальное количество предложений, наличие обязательных ключевых слов.
22. Если заданы ключевые слова, комментарий должен содержать хотя бы одно из них для прохождения валидации.
23. Комментарии не считаются валидными если они состоят только из одной цифры, одного короткого слова, одного символа, набора эмодзи или бессмысленного короткого текста.
24. Только валидные комментарии могут запускать игру, продлевать таймер, попадать в списки участников и участвовать в розыгрышах.


5. Антиспам
24. Необходимо ограничить частоту валидных комментариев от одного участника.
25. Ограничение должно настраиваться.
26. Например: один валидный комментарий от пользователя раз в X минут.
27. Если пользователь пишет чаще допустимого интервала, комментарий считается невалидным.
28. Невалидные по антиспаму комментарии не должны продлевать таймер, участвовать в игре и попадать в списки участников.


6. Призовые категории
29. Система должна поддерживать несколько типов победителей.
30. Главный победитель - последний валидный комментатор, после которого в течение периода тишины не появилось новых валидных комментариев.
31. Должна быть категория последних N участников.
32. Количество таких участников должно настраиваться.
33. Каждый из них может получать отдельный приз.
34. Должна быть категория первых N комментаторов.
35. Количество должно настраиваться.
36. Среди первых участников должен быть случайный розыгрыш приза.
37. Количество победителей и размер приза должны настраиваться.


7. Отображение игры на сайте
38. На странице игры должны отображаться: текущий статус игры, таймер игры, текущий последний валидный комментатор, список последних участников, список первых участников, призовые категории и условия выигрыша, правила участия.
39. Список первых участников фиксируется после достижения лимита.
40. Список последних участников обновляется динамически.
41. Все призовые категории должны быть видны одновременно.


8. Отображение невалидных комментариев
42. Система должна сохранять и валидные, и невалидные комментарии.
43. Невалидные комментарии должны отображаться на странице игры.
44. Невалидные комментарии должны быть визуально выделены (например красным цветом).
45. При нажатии на невалидный комментарий пользователь должен видеть причину отклонения.
46. Примеры причин: недостаточно слов, комментарий отправлен слишком часто, комментарий состоит только из цифр, комментарий состоит только из эмодзи.
47. Невалидные комментарии не должны продлевать таймер, участвовать в игре и попадать в списки победителей.


9. Завершение игры и результаты
48. После завершения игры система должна зафиксировать результаты.
49. На странице игры должны отображаться: главный победитель, победители среди последних участников, победитель случайного розыгрыша среди первых.
50. Результаты после завершения не должны изменяться автоматически.
10. Автоматические действия после завершения
51. Желательно предусмотреть возможность автоматического ответа победителям в комментариях.
52. Если автоответы не реализуются, система должна показывать список победителей, показывать ссылки на их комментарии и позволять администратору вручную уведомить победителей.


11. Админ‑панель
53. Администратор должен иметь возможность: привязать игру к YouTube‑видео, включить режим ожидания первого комментария, задать базовый таймер, задать продление таймера, задать период тишины, задать правила валидности комментариев, задать антиспам‑ограничения, задать количество первых и последних участников, задать призы и категории победителей, запускать и завершать игру.
12. Общая логика работы
54. Администратор настраивает игру заранее.
55. Игра ожидает первый валидный комментарий.
56. Первый валидный комментарий запускает игру.
57. Каждый следующий валидный комментарий продлевает таймер.
58. Сайт показывает ход игры в реальном времени.
59. После периода тишины игра завершается.
60. Система фиксирует победителей и публикует результаты.
